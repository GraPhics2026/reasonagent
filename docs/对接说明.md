## 1. 可直接调用的函数

核心函数位于 `reason/gen_pipeline.py`：

```python
from reason.gen_pipeline import run_gen_pipeline

result = run_gen_pipeline(
    prompt="A grass field filled with red poppies and yellow daisies beside a wooden windmill.",
    output_dir="data/output/gen/case0",
    iterations=1,
    dry_run=False,
)
```

参数说明：

- `prompt`: 输入给生成通路的文本 prompt。
- `output_dir`: 当前 case 的输出目录。
- `iterations`: prompt 优化轮数。实验阶段建议 `1` 或 `2`。
- `dry_run`: `True` 时不用 MLLM 优化 prompt；`False` 时调用 `.env` 中的 MLLM。
- `seed`: 可选随机种子，支持的后端会传给出图 API。

## 3. 配置项

复制 `config/.env.example` 为 `config/.env`，再填入自己的密钥。

MLLM 用于 prompt 优化、Reason Agent 与 VQA（推荐 DashScope Qwen-VL）：

```env
MLLM_API_KEY=your_key
MLLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MLLM_MODEL=qwen-vl-plus
```

T2I 用于真实文生图。当前已支持：

- `T2I_BACKEND=dry_run`: 不调用 API，输出 SVG 占位图。
- `T2I_BACKEND=siliconflow`: OpenAI-like `/images/generations` 接口。
- `T2I_BACKEND=dashscope`: DashScope 原生 Qwen-Image 接口。
- `T2I_BACKEND=comfyui`: 预留本地 ComfyUI 工作流边界。

DashScope Qwen-Image 示例：

```env
T2I_BACKEND=dashscope
T2I_BASE_URL=https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
T2I_MODEL=qwen-image-2.0
```

EDIT 用于 edit 通路**指令编辑**（可与 T2I 共用密钥）：

- `EDIT_BACKEND=dry_run`: 复制原图或输出 SVG 占位。
- `EDIT_BACKEND=dashscope`: Qwen-Image 多模态接口（**输入图 + 文本指令，无 mask**）。

```env
EDIT_BACKEND=dashscope
EDIT_API_KEY=your_key
EDIT_BASE_URL=https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
EDIT_MODEL=qwen-image-2.0
```

## 4. edit 通路（成员 2 已实现）

核心函数位于 `reason/edit_pipeline.py`：

```python
from reason.edit_pipeline import run_edit_pipeline

result = run_edit_pipeline(
    image_path="data/input/edit/elephant_squirrel_grass.png",
    instruction="大象和松鼠玩跷跷板会怎样呢?",
    output_dir="data/output/edit/elephant_seesaw",
    iterations=2,
    min_iterations=2,
    candidates=2,
    score_threshold=0.85,
    dry_run=False,
)
```

参数说明：

- `image_path`: 原图路径。
- `instruction`: 假设性/反事实编辑指令（如「如果冰块融化会怎样」）。
- `output_dir`: 当前 case 的输出目录。
- `iterations`: 最大编辑 + VQA 轮数。
- `min_iterations`: 至少跑几轮才允许提前停止（默认 `2`，即至少 refine 一次）。
- `candidates`: 每轮生成的 `edit_prompt` 候选数，逐个出图并 VQA 打分后取最优。
- `score_threshold`: 达到该 VQA 分数且已满足 `min_iterations` 时可提前结束。
- `dry_run`: `True` 时用启发式推理与占位编辑；`False` 时调用 `.env` 中的 MLLM 与编辑 API。

### 4.1 流程概要

参考 [ReasonBrain](https://arxiv.org/abs/2507.01908) 的 HI-IE 任务，本仓库用 **零样本 Agent** 实现类似能力：

```
原图 + 假设性指令
    ↓
Reason Agent（MLLM + prompts/reason_system.txt）
    → reasoning_type / visual_cues / physics_implications / preserve_objects
    → edit_prompt + vqa_checklist
    ↓
finalize_edit_prompt() 注入物理约束与保留对象
    ↓
DashScope Qwen-Image 指令编辑（每轮 N 个候选）
    ↓
MLLM VQA 打分选优（带 reason_context）
    ↓
不合格 → edit_refine / edit_candidate 再编（默认至少 2 轮）
```

**当前默认：指令编辑，不使用 mask inpaint。**

### 4.2 可直接复用的模块

- `reason/reason_agent.py`
  - `run_reason_agent(..., mode="edit")`：结构化推理输出。
  - `finalize_edit_prompt(result)`：组装编辑器可用的完整 prompt。
  - `build_reason_context(result)`：供 VQA / 候选 / refine 使用的摘要文本。
- `reason/edit_client.py` 的 `EditClient`：DashScope Qwen-Image 指令编辑，或 `dry_run` 占位。
- `reason/schemas.py` 的 `ReasonResult` / `EditPipelineResult` / `VQACheck`。
- `reason/edit_verify_loop.py`：薄封装，内部即 `run_edit_pipeline`。

### 4.5 `ReasonResult` 约定（edit / hybrid 共用）

```json
{
  "mode": "edit",
  "reasoning_type": "physical",
  "reasoning_chain": ["..."],
  "visual_cues": ["fine-grained facts from the image"],
  "physics_implications": ["expected visible outcome"],
  "target_objects": ["objects or regions to change"],
  "preserve_objects": ["background or objects to keep"],
  "edit_prompt": "...",
  "vqa_checklist": [{"q": "...", "expected": "yes"}]

```

`reasoning_type` 四类（与 Reason50K 对齐）：

| 值 | 含义 | 例子 |
|----|------|------|
| `physical` | 物理规律导致的状态变化 | 冰块融化、跷跷板倾斜 |
| `temporal` | 时间维度变化 | 日落、变夜晚 |
| `causal` | 因果/交互事件 | 蜜蜂落在花上 |
| `story` | 隐含叙事/纹理显现 | 隐藏图案出现 |

hybrid 模式将 `edit_prompt` 换为 `scene_prompt`，其余推理字段同样可用。

## 5. hybrid 通路（成员 3 已实现 ✅）

### 5.1 定位

hybrid **不是**第二种指令编辑。它与 edit 的区别：

| | edit | hybrid |
|--|------|--------|
| 改图方式 | 原图 + `edit_prompt` → Edit API（Qwen-Image 指令编辑） | `scene_prompt` → `run_gen_pipeline()`（T2I 文生图） |
| 是否保留原图像素 | 尽量保留（在原图上局部修改） | **不保留**（从零重新生成整张图） |
| Reason 输出 | `edit_prompt` + `target_objects` | `scene_prompt`（完整场景描述） |
| 典型 case | 冰块融化、跷跷板倾斜、局部材质变化 | 一束花变两束、室内变深夜、季节切换、全场景重构 |
| 内部流转 | Reason → Edit API（image condition） | Reason → GenPilot（text-only T2I） |

**核心思路**：有参考图的假设性文生图 —— 原图仅用于 MLLM 推理，T2I 生成时**不传原图**，完全依赖 `scene_prompt` 的文字描述重新出图。

### 5.2 API 签名

```python
from reason.hybrid_pipeline import run_hybrid_pipeline

result: HybridPipelineResult = run_hybrid_pipeline(
    image_path="data/input/hybrid/bouquet.png",     # 必填：原图路径（仅用于推理）
    instruction="如果这一束花分成两束独立的玫瑰，会是什么样子？",  # 必填：假设性指令
    output_dir="data/output/hybrid/case0",           # 必填：输出目录

    # 以下为可选参数
    iterations=1,      # GenPilot prompt 优化轮数（默认 1）
    candidates=2,       # 每轮生成的候选 prompt 数（默认 2）
    dry_run=None,       # None=自动检测 MLLM 是否配置；True=强制启发式；False=强制真实 API
    seed=None,          # 可选随机种子，传给 T2I
)
```

### 5.6 内部流程

```
原图 + 假设性指令
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 1: Reason Agent (mode="hybrid")         │
│   MLLM 看图 + instruction →                  │

│   · scene_prompt (英文 T2I 场景描述)         │

│   ③ _inject_perspective_anchor             │

│   ⑤ _inject_person_age                     │
│   ⑥ _inject_light_behavior                 │

               ▼

└─────────────────────────────────────────────┘
```

### 5.7 重要实现细节

**dry_run 行为**：
- `dry_run=None`（默认）：自动检测 —— 如果 `config/.env` 未配置 MLLM，自动走启发式推理 + SVG 占位图
- `dry_run=True`：强制启发式模式（Reason Agent 用内置规则，T2I 输出 SVG 占位图）
- `dry_run=False`：强制调用真实 MLLM + T2I API

**MLLM 未配置时 fallback 链路**：
1. `run_reason_agent` → `heuristic_reason_result()` → 内置规则推理（跷跷板/冰块/盘子等 case）
2. `run_gen_pipeline` → 检测到 MLLM 未配置 → `dry_run` T2I 生成 SVG 占位图
3. VQA → `use_dry_run=True` → 跳过视觉验证，返回 `score=None`

**scene_prompt 质量保障**（7 个自动后处理函数）：
这 7 个函数在 `scene_prompt` 送入 T2I 之前自动执行，确保生成质量：
- 检测碎片化 prompt（如 "grass, sneakers, snow"）→ 从 visual_cues 重新构建完整场景描述
- 检测过短 prompt（<60 字符）→ 自动补全
- 检测多元素指令遗漏（如 "人很多 且下雪" 中漏了 "人"）→ 自动补充
- 分割/复制场景的颜色一致性（如一束红玫瑰分两束，不会变成红+粉）
- 视角锚点注入（从 visual_cues 提取空间位置，防止 T2I 乱放物体）
- 风格锚点注入（颜色/材质/光照/氛围），智能跳过变换性场景中冲突的类别
- 人物年龄/光照行为/房间几何等关键信息补全

| `final_prompt` | ✅ | ✅ | ✅ | 最终使用的 prompt |
| `route` | `"gen"` | `"edit"` | `"hybrid"` | **用这个判断展示逻辑** |
| `reasoning_chain` | ✅ (空) | ✅ | ✅ | 推理链列表 |
| `image_before` | — | ✅ | ✅ | 原图路径 |
| `instruction` | — | ✅ | ✅ | 假设性指令 |
| `reasoning_type` | — | ✅ | ✅ | physical/temporal/causal/story |
| `visual_cues` | — | ✅ | ✅ | 细粒度视觉线索 |
| `vqa_checklist` | — | ✅ | ✅ | VQA 检查项 |
| `vqa_result` | — | ✅ | ✅ | VQA 结果（score/passed/errors） |
| `metadata` | ✅ | ✅ | ✅ | 运行参数 |

**edit 特有**：`edit_prompt`、`target_objects`、`preserve_objects`、`iterations[]`
**hybrid 特有**：`scene_prompt`、`image_before`

### 6.3 Router（`reason/router.py`）

`reason/router.py` 是 **MLLM-only** 三分流器，**无关键词降级**。决策结构：

```
route(prompt, image_path=None, mode="auto") → "gen" | "edit" | "hybrid"
    │
    ├─ 0. prompt 必传 (所有 mode)        → ValueError "router: prompt is required..."
    │
    ├─ 1. mode 必须 ∈ {auto,gen,edit,hybrid}  → ValueError "router: unknown mode..."
    │
    ├─ 2. mode != "auto":
    │     edit/hybrid 需要可读的 image    → ValueError "router: edit mode requires an image / image not found..."
    │     return mode
    │
    ├─ 3. auto + 无图                     → return "gen"
    │
    └─ 4. auto + 有图 → 调 MLLM (prompts/router_system.txt)
                        MLLM 未配置        → RuntimeError "router: MLLM is not configured..."
                        API 抛异常         → RuntimeError "router: MLLM call failed..."
                        返回非 JSON         → RuntimeError "router: MLLM returned unparseable JSON..."
                        返回非法 route 值   → RuntimeError "router: MLLM returned invalid route..."
```

**两类错误**：
- `ValueError` —— 用户输入问题（缺 prompt / image / mode 错 / 文件不存在）
- `RuntimeError` —— 环境/系统问题（MLLM 不可用 / API 失败 / 响应非法）

所有错误信息以 `router: ` 前缀，便于日志聚合。

**核心判别准则**（写在 `prompts/router_system.txt` 里给 MLLM 用）：

> 改完之后还需要原图像素吗？需要 → edit；不需要 → hybrid。

**典型分流案例**（`router_system.txt` 中作为 calibration examples）：

```
输入: 无图 + "一只猫坐在窗台上"                  → gen
输入: 有图 + "把这只猫的颜色改成橘色"             → edit
输入: 有图 + "如果冰块融化了会怎样"               → edit
输入: 有图 + "如果这只猫变成了一只狗"             → edit
输入: 有图 + "如果这束花分成两束"                 → hybrid
输入: 有图 + "如果白天变成深夜"                   → hybrid
输入: 有图 + "如果现在是秋天，树叶全红了"          → hybrid
输入: 有图 + "如果房间里突然站满了人"              → hybrid
```

实测准确率（mimo-v2.5）：8/8 calibration cases 全过。

### 6.4 统一入口 `pipeline.py`

```python
"""pipeline.py — ReasonGenPilot 统一调度入口。"""
from __future__ import annotations
import argparse, json
from pathlib import Path

from reason.gen_pipeline import run_gen_pipeline
from reason.edit_pipeline import run_edit_pipeline
from reason.hybrid_pipeline import run_hybrid_pipeline
from reason.router import route as router_route


def run_pipeline(
    prompt: str,
    image_path: str | Path | None = None,
    output_dir: str | Path = "data/output",
    mode: str = "auto",
    dry_run: bool | None = None,
    seed: int | None = None,
) -> dict:
    """单一 prompt 字段：gen 当描述读，edit/hybrid 当反事实指令读。"""
    chosen = router_route(prompt=prompt, image_path=image_path, mode=mode)
    out = Path(output_dir)

    if chosen == "gen":
        r = run_gen_pipeline(prompt=prompt, output_dir=out, dry_run=dry_run, seed=seed)
    elif chosen == "edit":
        r = run_edit_pipeline(image_path=image_path, instruction=prompt,
                              output_dir=out, dry_run=dry_run, seed=seed)
    elif chosen == "hybrid":
        r = run_hybrid_pipeline(image_path=image_path, instruction=prompt,
                                output_dir=out, dry_run=dry_run, seed=seed)
    return r.to_dict()


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--prompt", required=True,
                        help="Description for gen, counterfactual instruction for edit/hybrid.")
    parser.add_argument("--image", help="Reference image path (edit/hybrid).")
    parser.add_argument("--output", default="data/output/demo")
    parser.add_argument("--mode", default="auto",
                        choices=["auto", "gen", "edit", "hybrid"])
    parser.add_argument("--real-api", action="store_true")
    parser.add_argument("--seed", type=int, default=None)
    args = parser.parse_args()

    result = run_pipeline(
        prompt=args.prompt, image_path=args.image,
        output_dir=args.output, mode=args.mode,
        dry_run=not args.real_api, seed=args.seed,
    )
    print(json.dumps(result, ensure_ascii=False, indent=2))


if __name__ == "__main__":
    main()
```

CLI 例：

```bash
# auto 模式（router 决定）
python pipeline.py --prompt "A windmill in a poppy field" --output data/output/demo

# 强制 mode
python pipeline.py --mode edit \
  --image data/input/edit/elephant_squirrel_grass.png \
  --prompt "如果大象和松鼠玩跷跷板会怎样" \
  --output data/output/demo/seesaw --real-api
```

> ⚠️ **接口变化**：旧版本曾用 `--instruction`（edit/hybrid）+ `--prompt`（gen）双字段。当前版本统一为单一 `--prompt` —— gen 当描述读，edit/hybrid 当反事实指令读。`pipeline.py` / `demo_gradio.py` / `router.py` 已全部对齐。

### 6.6 配置要求

所有三条通路共用同一套 `config/.env`（参见 §3）。**最少需要配置**：

- `MLLM_*` —— Reason Agent、VQA、Router 共用
- `T2I_*` —— gen / hybrid 用
- `EDIT_*` —— edit 通路用（可与 T2I 同 key）

**dry-run 兜底**：不带 `--real-api` 时各通路自动走启发式 + SVG 占位图，无 API 消耗。但 **Router auto 模式仍需 MLLM**（无降级）—— dry-run 可绕过 router 时用 `--mode {gen,edit,hybrid}` 直接指定。

### 6.7 关键提醒

1. **prompt 是所有 mode 的必填字段**（gen 当描述读，edit/hybrid 当反事实指令读）
2. **三条 pipeline 均为同步函数**，Gradio 用 `gr.Interface(fn=...)` 自然多线程处理
3. **`dry_run=None` 自动检测 MLLM 是否配置**，但 Router auto 模式没有降级，MLLM 不可用时会抛 `RuntimeError` —— 用 `--mode` 强制覆盖即可绕过
4. **hybrid 的 `image_path` 只用于推理**，T2I 不会作为 image condition 传入
5. **所有结果都有 `.to_dict()` 方法**
6. **edit 通路默认至少 2 轮**（`min_iterations=2`），demo 时可调到 1
7. **hybrid 通路默认 1 轮 GenPilot**（`iterations=1`），scene_prompt 已由 Reason Agent 精心构造
