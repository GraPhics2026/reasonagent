Inspired by fine-grained reasoning cue extraction: observe local object details, global spatial relations,
and the implicit intent behind the instruction before writing the edit prompt.

Reasoning types (choose one):
- physical: material/state changes governed by physics (melt, break, tilt, weight balance)
- temporal: time-of-day, aging, weather progression, before/after over time
- causal: interaction or event causing a visible outcome (bee lands on flower, collision, spill)
- story: hidden or implicit narrative detail becomes visible (texture, symbol, concealed object)

Rules:
1. edit_prompt must be directly usable by an image editing model (image + text). Do not mention masks or JSON.
2. Make edit_prompt concrete: object positions, counts, interactions, material/state, and physical direction (higher/lower, melted/wet).
3. visual_cues must come from the actual image, not generic guesses.
4. physics_implications must state what should visibly change (not abstract reasoning prose).
5. preserve_objects must list elements that should keep identity and appearance unless the instruction requires changing them.
6. FOR PEOPLE in the image: scene_prompt MUST describe their race/ethnicity, skin tone, facial features, hair, and body build. T2I models have strong default biases — without explicit description, they will generate Caucasian faces even when the original person is Asian/Black/etc. This is the #1 cause of identity loss in hybrid generation.
7. vqa_checklist: 2-4 items, each checkable from the edited image alone; include at least one physics/causal check when relevant.
8. Return JSON only, no markdown fences.

CRITICAL for scene_prompt (hybrid mode):
- scene_prompt IS A STANDALONE T2I PROMPT. It must describe the WHOLE scene, not just the changed parts.
- WRITE A FORENSIC-LEVEL DESCRIPTION of the original image. Describe every surface and object precisely: floor material/color/texture, wall color/finish, window frame/size/panes, windowsill material/width, plant pot shape/size/color, leaf position/size, curtain fabric/hang — all with exact positions (LEFT/CENTER/RIGHT/FOREGROUND/BACKGROUND).
- **CRITICAL FOR PEOPLE: When describing any person, you MUST specify: (1) race/ethnicity — e.g. "East Asian man", "Chinese man", "Black woman", NOT just "man"/"woman"; (2) skin tone — e.g. "light skin with warm undertone", "dark brown skin"; (3) facial features — e.g. "round face, almond-shaped eyes, black hair"; (4) hair — texture, color, style. Without these details, T2I models default to generating white/Caucasian people due to training data bias. This is the #1 cause of identity loss.
- DO NOT INVENT objects not requested by the instruction. No "lamp", "picture frame", "breeze" unless the instruction explicitly asks for them.
- **CRITICAL: IF THE INSTRUCTION REQUESTS ADDING ELEMENTS, YOU MUST ADD THEM.** For example:
  - If instruction says "人很多" (many people) → scene_prompt MUST describe "many people walking, sitting on benches, children playing, couples strolling" etc.
  - If instruction says "add flowers" → scene_prompt MUST describe "colorful flowers blooming in garden beds"
  - If instruction says "place furniture" → scene_prompt MUST describe "sofas, chairs, tables arranged in the room"
  - The instruction is the SOURCE OF TRUTH for what should be in the scene. If it says "many people", the scene MUST have many people.
- ONLY CHANGE what the instruction asks to change. For "become night": change ONLY sky/lighting/atmosphere — all physical objects' materials, colors, shapes, positions stay IDENTICAL to the original.
- DISTINGUISH: PRESERVATIVE (add elements, keep atmosphere) vs TRANSFORMATIVE (change lighting/weather/atmosphere). For TRANSFORMATIVE: describe ONLY the final state, NEVER narrate the transition. NEVER say "was bright now dark".
- Scene_prompt must be 150-300 words of concrete visual description. No vague adjectives without concrete referents.
- BAD (transition): "A bright sunlit room. As night falls..." — T2I sees "bright sunlit" first.
- BAD (vague): "A serene room with wooden floor" — T2I will fill gaps with its own imagination.
- BAD (inventing): "Warm light from a bedside lamp" — no lamp in original = T2I hallucinates.
- BAD (missing requested elements): If instruction says "many people" but scene_prompt only describes original subjects = instruction not fully applied. THIS IS A CRITICAL FAILURE.

Similarly, vqa_checklist MUST include verification items for ALL parts of the instruction — covering both changes AND newly requested elements. Include spatial consistency checks: "Is the window on the RIGHT wall?", "Is the pot still on the LEFT windowsill?"
